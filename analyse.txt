海流粒子图绘制逻辑分析
=====================

一、入口与触发位置
------------------

海流粒子图不是独立的渲染层，而是挂在 ncdfOverlayFactory 的总渲染流程里。

主入口在 [src/ncdfoverlayfactory.cpp](src/ncdfoverlayfactory.cpp) 的 DoRenderncdfOverlay：

- 当 m_bShowCurrentDir 为真时，先绘制流向箭头。
- 当 m_bShowParticles 为真时，调用 RenderParticles(vp)。
- 当粒子关闭时，调用 ClearParticles()，立即释放粒子状态。

也就是说，粒子图是否出现，完全由插件状态开关控制，绘制和更新都在同一个渲染流程里完成。

二、粒子数据结构
----------------

粒子状态定义在 [src/ncdfoverlayfactory.cpp](src/ncdfoverlayfactory.cpp) 底部：

- Particle 保存单个粒子的生命周期、历史位置和当前推进节奏。
- ParticleNode 保存每个历史点的经纬度、屏幕坐标和颜色。
- ParticleMap 是粒子容器，内部持有 std::vector<Particle>，并缓存上一次 viewport，用于视口变化时重算屏幕坐标。

核心历史结构是一个固定长度环形缓冲区：

- m_History[ ] 存储轨迹点。
- m_HistoryPos 指向当前写入位置。
- m_HistorySize 表示当前有效历史长度。
- m_Run 控制并非每帧都推进，而是按节奏分步更新，避免粒子移动过快。
- m_Velocity 用于后续轨迹长度调节，速度越高，尾迹越长。

三、更新逻辑
------------

RenderParticles 的第一部分是更新粒子位置，它依赖当前网格数据：

- 数据源来自 gui->gridu 和 gui->gridv。
- 只有在 ni > 1 且 nj > 1 时才继续。
- 当前数据边界由 myMessage.firstGridPointLat / Long 和 lastGridPointLat / Long 提供。

更新流程可以拆成四步：

1. 计算粒子数量和历史长度

- 粒子密度来自插件的 m_iParticleDensity。
- 代码使用 density = 0.2 * sliderVal。
- history_size 按 27 / sqrt(density) 计算，再限制在 4 到 MAX_PARTICLE_HISTORY 之间。
- 总粒子数按 density * ni * nj 计算，并限制上限，避免渲染爆炸。

2. 视口变化时重算屏幕坐标

- 如果缩放、旋转、中心点变化，就遍历所有粒子历史点。
- 对每个历史经纬度调用 GetCanvasPixLL，刷新 m_Screen。
- 这样做的目的是避免每帧重复做完整坐标转换。

3. 推进已有粒子

- 只有 m_bUpdateParticles 为真，或者数据变化后的 burst 计数还在时，才推进粒子。
- 每个粒子不是每一帧都更新，而是先通过 m_Run 做节流。
- 粒子从当前历史点取经纬度，再用 getInterpolatedValue 从 u/v 网格插值得到当前位置的流速分量。
- 若值有效，则计算流速模长和方位角，再用球面航迹公式推进下一位置。

推进公式本质上是按球面距离和航向在经纬度上走一步：

- 先把速度换算成距离增量 d。
- 再计算弧度角 angr、latr、D。
- 通过 sin/cos 和 asin 组合更新经纬度。

4. 生成新粒子

- 当粒子数量不足目标值时，会随机在数据范围内撒点。
- 每个候选点会检查插值速度是否有效，并优先保留有流速的区域。
- 新粒子会预填充历史轨迹，让刚出现时就有完整尾迹，而不是先从一个点慢慢长出来。

四、绘制逻辑
------------

粒子真正绘制时，采用 OpenGL 线段批量渲染，只有在 m_pdc 为空时才走这条路径，也就是 OpenGL 渲染分支。

绘制步骤如下：

1. 开启抗锯齿和混合

- glEnable(GL_LINE_SMOOTH)
- glEnable(GL_BLEND)
- glBlendFunc(GL_SRC_ALPHA, GL_ONE_MINUS_SRC_ALPHA)
- glLineWidth(2.3f)

2. 遍历每个粒子的历史链

- 从 m_HistoryPos 开始逆序遍历历史点。
- 颜色直接使用历史点里缓存的颜色值。
- 透明度从 250 开始逐步衰减。
- 每段线都用当前历史点和前一个历史点之间的插值位置组成。

3. 速度影响尾迹长度

- 速度越大，trailSteps 越大，alpha 衰减越慢，尾迹越长。
- 速度越小，alpha 衰减越快，尾迹越短。
- 这样能让快流区域更显眼，慢流区域更收敛。

4. 逐段画线

- 对每一对相邻历史点，调用 glBegin(GL_LINES) / glEnd() 画线段。
- 线段颜色使用粒子当前历史点的 RGB，透明度由 alpha 控制。
- 如果历史点超出屏幕宽度或无效，就跳过。

五、刷新节奏
------------

粒子动画不是依赖连续重绘，而是靠定时器驱动：

- 构造函数里绑定 wxTimer。
- 每次定时器触发后，把 m_bUpdateParticles 设为真，并请求画布刷新。
- RenderParticles 结束时，如果定时器没跑，就启动一个 50ms 的 one-shot 定时器。

这意味着粒子图是“按帧更新、按帧重绘”的，刷新节奏由粒子系统自己维持。

六、和箭头绘制的区别
--------------------

海流箭头和粒子图共享同一份 u/v 数据，但绘制方式不同：

- 箭头是离散采样，在固定间隔位置画方向箭头。
- 粒子图是连续轨迹，靠插值和历史缓存形成流线效果。
- 箭头更适合读方向，粒子更适合看流场结构和速度变化。

七、这段代码的实际效果
----------------------

从实现上看，粒子图的核心目标不是精确数值展示，而是把海流场做成可视化“流动感”：

- 用插值保证粒子沿着连续流场移动。
- 用历史环形缓冲区保存尾迹。
- 用速度控制尾迹长度和透明度衰减。
- 用定时器把更新节奏固定下来，形成持续运动的视觉效果。

如果后续要继续优化，这个实现的关键观察点是：粒子更新、历史长度和透明度衰减都集中在 RenderParticles 里，修改动画风格基本只需要改这个函数。
